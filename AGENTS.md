# XFile 仓库协作约定

## 提交与更新日志

- 每次创建 Git 提交前，必须同步更新 `docs/更新日志.md`；任何代码、配置、依赖、数据库、文档或用户可见行为的改动都不能例外。
- 更新日志按日期倒序维护，说明本次提交的新增、变更、修复、兼容性影响及已执行的验证。
- “关于 XFile”页面必须继续优先展示 `docs/更新日志.md`，不得移除其优先排序与默认选中逻辑。
- 项目许可证为 `AGPL-3.0-only`；除非用户明确要求，不得替换、删除或弱化根目录 `LICENSE` 与 `docs/授权说明.md` 中的许可说明。
- Markdown 与源码统一使用 UTF-8 编码。
